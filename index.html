正则表达式

JS中的用法

常见使用方法

var reg = /\bis\b/g;
var reg = new RegExp('\\bis\\b','g'); //注意转义

表达式后面的g代表修饰符:

  • g: global全文搜索，搜索带第一个匹配停止。
  • i: ignore case 忽略大小写，默认大小写敏感。
  • m:multiple lines 多行搜索

比较常见的情况是用在replace函数中的第一个参数。

字符

原义文本字符

代表字符本身。 /a/g，匹配所有a。

元字符

元字符是正则表达式中有特殊含义的非字母字符：

      • ？ $ ^ . | \ ( ) { } [ ]
  • \t 水平制表符 //例如ab\t 代表 ‘ab’ + tab
  • \v 垂直制表符
  • \n 换行符
  • \r 回车符
  • \0 空字符
  • \f 换页符

我们可以使用[]来构建一个简单的类，所谓类是否和某些特性的对象，一个泛指，不是特指某个字符。

字符类

例如[abc]匹配的是包含abc中任意一个的字符:

'a1b1c1d1'.replace(/[abc]/g, 'X')
//'X1X1X1d1'

[^abc]在前面加入^代表取反，匹配不是abc的字符。

范围类

[a-z]表示从a-z的任意字符（闭区间，即包含a和z本身）。

[a-zA-Z0-9] 区间可以直接连写。

[0-9-] 代表匹配0到9和’-‘。

预定义类

正则表达式中预定义的一些类，可以简化书写

  • . == [^\r\n] 除了回车和换行符之外的所有字符。
  • \d == [0-9] 数字字符。
  • \D == [^0-9] 非数字字符。
  • \s == [\t\n\x0B\f\r] 空白符。
  • \S == [^\t\n\x0B\f\r] 非空白符。
  • \w == [a-zA-Z_0-9] 单词字符（字母、数字下划线）
  • \W == [^a-zA-Z_0-9] 非单词字符。
  • ^ 以xxx开始。
  • $ 以xxx结束。
  • \b 单词边界 \B 非单词边界。

例子：
‘This is a boy’.replace(/is/g, ‘0’) // ‘Th0 0 a boy’
‘This is a boy’.replace(/\bis\b/g, ‘0’) // ‘This 0 a boy’
‘This is a boy’.replace(/is\b/g, ‘0’) // ‘Th0 is a boy’

'@123@abc@'.replace(/@./g, 'Q') // 'Q23Qbc@' 匹配了@加上任意字符。
'@123@abc@'.replace(/^@./g, 'Q') //'Q23@abc@' 加上^匹配以@加上任意字符开头的字符
'@123@abc@'.replace(/.@/g, 'Q') //'Q23@abc@' 匹配不是开头的@加上任意字符。
'@123@abc@'.replace(/.@$/g, 'Q') //'Q23@abc@' 加上$匹配以@加上任意字符结尾的字符

量词

如果要连续匹配20个数字: \d{20}

  • ? 出现零次或一次（最多出现一次）。
    • 出现一次或多次（最少出现一次）.
    • 出现零次或多次（任意次）
  • {n} 出现n次。
  • {n,m} 出现n到m次。
  • {n,} 至少出现n次。

贪婪和非贪婪

12345678 对应 \d{3,6} 这时候就要做出选择到底匹配3个还是6个。

默认情况下正则是尽量匹配多的，所有会匹配到123456。

如果要切换成非贪婪模式： \d{3,6}? 这样匹配的就是123。

分组

如果要匹配this出现3次的字符串。 this{3}并不能实现，而是匹配thi加3个s.
正确的情况(this){3} 加上括号，成立分组。

或

this|that 这样等于this或者that, 也就是说|是直接把正则表达式的前后分开。

asd(this|that)cda 这样中间就成立了一个组。

'asdthiscdaasdthatcda'.replace(/asd(this|that)cda/g, X) // XX

反向引用

需求 2015-12-25 => 12/25/2015 -变成/ 但是难点是顺序也变了。

'2015-12-25'.replace(/(\d{4})-(\d{2})-(\d2)/g, '$2/$3/$1') // 12/25/2015

$加数字，可以按顺序捕获分组。如果不想捕获某个分组前面加?:例如(?:\d{2})。

前瞻后瞻

正则表达式从文本头部向尾部解析，所以文本尾部方向就是向前。

前瞻就是正则表达式在向前匹配规则的时候，检查是否符合断言。符合的叫正向匹配，不符合的叫负向匹配。例如向前匹配符合的，就像正向前瞻。（JS不支持后瞻，也就是说逆向查找）。

断言则是为正则表达式加上限制条件。
例如：\w(?=\d) 匹配一个字母，并且其后面是数字。

'a2*3'.replace(/\w(?=\d)/g, 'X') // X2*3

'a2*34v8'.replace(/\w(?=\d)/g, 'X') // X2*X4X8

'a2*34v8'.replace(/\w(?!\d)/g, 'X') // aX*3XvX

正则表达式的对象属性

global、ignoreCase、multiline 默认都是false；

lastIndex 当前表达式匹配内容的最后一个字符的下一个位置；

source 正则表达式的文本字符串

var reg1 = /\w/ ;
var reg2 = /\w/gim;
console.log(reg1.global) //false;
console.log(reg2.ingoreCase) //true;
console.log(reg2.multiline) //true;
这三个属性都是只读了，只能在生成的时候设置，不能后续设置。
console.log(reg1.source) // '\w'
console.log(reg2.source) // '\w'

正则表达式自带的方法

test : 用于测试字符串参数中是否存在匹配正则表达式模式的字符串。
var reg1 = /\w/ ;
var reg2 = /\w/g ;
reg1.test(‘a’) //true;
reg2.test(‘ab’) //true;
reg2.test(‘ab’) //true;
reg2.test(‘ab’) //false; 执行三次后变成false

为什么加上g之后执行第三次的时候变成false，是因为lastIndex属性变了，每次执行lastIndex都在变。所以为了避免这个问题，一般不加g标志。

exec: 使用正则表达式模式对字符串进行搜索，并将更新全局对象RegExp的属性以发音匹配结果（lastIndex）。

如果没有匹配的文本则返回null，否则返回一个结果数组：

  • index 声明匹配文本的第一个字符的位置。
  • input 存放被检索的字符串(匹配到的字符串，子表达式相匹配的文本)。

例子：

var reg3 = /\d(\w)(\w)\d/;
var reg4 = /\d(\w)(\w)\d/g;

var ts = '1az2bb3cc4dd5ee';
var ret = reg3.exec(ts);

//在没有全局的时候
console.log(reg3.lastIndex, ret.index, ret.toString());
//0  1  1az2,a,z    
console.log(reg3.lastIndex, ret.index, ret.toString());
//0  1  1az2,a,z    （lastIndex没有变）

while(ret = reg4.exec(ts)){
    console.log(reg4.lastIndex, ret.index, ret.toString());
}
// 5 1 1az2,a,z
// 11 7 3cc4,c,c

常见的字符串方法配合正则

string.prototype.search(reg)

用于检索字符串中是否有匹配的字符串，返回第一个匹配结果的index，如果无则返回-1。注意search方法会忽略g标签，只查找到第一个。

'a1b2c3d1'.search(/\D/g) // 0;

string.prototype.match(reg)

区别在于g标志。

没有g标志时，只执行一次匹配，如果没有，返回null。如果有，返回数组，其中存放了它找到的有关信息[匹配的文本，字表达式匹配的文本] 有index和lastindex属性。与exec很像。

如果有g标志，区别在于返回的数组只有匹配到的元素，没有字表达式文本和index、input属性了。

string.prototype.split(reg)

用于较为复杂的分割情况：
‘a1b2c3d4e’.split(/\d/g) // [‘a’,’b’,’c’,’d’,’e’]

string.prototype.replace(reg, replacestr)

一种特殊的用法。

string.replace(reg, func())

这里的func有四个参数：

  • 匹配的字符串
  • 字表达式（分组内容），没有分组则没有参数，有几个分组就有几个参数
  • 匹配项在字符串中的index
  • 原字符串

需求 ‘a1b2c3d4’ => ‘a2b3c4d5’

'a1b2c3d4e5'.replace(/\d/g, function(match, index, origin)
{
    console.log(index);
    return parseInt(match)+1
}
)
//1 3 5 7 9 'a2b3c4d5e6'

'a1b2c3d4e5'.replace(/(\d)(\w)(\d)/g, function(match, index, group1, group2, origin)
{
    console.log(match);
    return group1+group3 //这样就把中间的(\d)去掉了
}
)
// '1b2'  '3d4' 
// 'a12c34e5'

我的JS学习笔记（一）

我的JS学习笔记一

1.函数相关

  • 多个脚本或函数在调用时一定要通过依赖关系注意引入的顺序。
  • 作用域：即函数或变量的有效范围。限定了那部分代码可以访问该变量或方法。
  • 一般来说稍微避免全局变量和函数的创建，因为其很容易被改变，可以通过一些方法来减少这种风险，比如闭包。
  • 函数内变量的生存期是仅在函数运行期间，运行结束后就将被JS的垃圾回收机制回收。全局变量的生存期就是页面的生存期。
  • 标识符：即变量的名称，JS的查找机制一般是先从作用域内查找，如无，即从下一个更大的作用域查找，以此内推。因而常在函数内申明变量，即为了减小JS查找标识符的内存消耗。

    2.关于对象

  • 对象有对应的属性和方法。例如车，颜色是属性，加速是方法等。JS中如Data对象，getTime()即是他的方法，能够计算调用时到1970年1月1日0点的毫秒数。
  • 创建对象。例如 var mydate = new Date();其中new告诉JS创建一个新对象，Date()即使Date对象的构造函数。
  • 注意数据存储时(特指对象等，和基本的文本，数字不同，文本数字是直接存储的数据)，并不是在变量中存储了数据，而是在变量中存储了这个数据（对象）的内存地址。例如当一个对象有两引用时，通过一个改变对象里的值，另一个也会改变，因为他们同时引用相同的地址。

内置对象常用方法

字符串

  • indexOf(),从前面开始找，lastIndexOf(),从后往前开始找，也可以理解为最后出现的位置,且两个方法第二个参数都可以输入开始查找的位置（也可以用在数组上）：

    "abcda".indexOf(a) //0

    "abcda".lastIndexOf(a) //4

  • substring(),两个参数，第一个为起始位置，第二个结束位置，substr()，第一个起始位置，第二个是截取长度，两个方法都是返回一个字符串。

  • toLowerCase()/toUpperCase()，转换大小写。
  • charAt(),接受一个参数，返回这个索引位置的字符，charCodeAt(),是返回特定位置的Unicode码，比如A的编码是65，fromCharCode(65)返回A，跟charCodeAt()正好相反。
  • trim(),去掉字符串两端的空格。

数组

  • concat(),连接两个单独的数组，返回一个新数组。
  • slice()，返回一段数组，接受两个参数，为开始索引和结束索引，返回的数组包含前者不包含后者。
  • join(),将所有元素做为字符串返回，接受的参数为中间可以间隔的元素。
  • sort(),将数组按照元素的unicode码排序，大写字母的序列在小写字母序列的前面。
  • reverse(),数组反序。
  • 循环函数：

测试型：every();some();filter();操作型：forEach();map().

var number = [1,2,3,4,5];
function isLessThan3(value,index,array)
{
var resultValue = false;
if(value < 3) returnValue = true;
return returnValue;
} 
alert(number.every(isLessThan3));

值得注意的是isLessThan3这个函数接受的三个参数，都是必须的，此为固定格式，在所有的数组循环函数中都是相同的。
every()要求数组中每个函数都要满足限定的条件，最终才会返回true，显然这里返回的是false;换成some(),就会返回true，因为它只要求有元素满足就可以。filter()则是取出这些元素，最后以一个数组形式返回。

function doubleNumber(value,index,array)
{
    var result = value * 2;
}
number.forEach(doubleNumber);

这样就将数组中的每个元素加倍了。但是forEach()没有返回值，map()跟forEach()的作用相同，但是在操作数组的同时会复制一份结果数组来作为返回值。

Math对象

Math对象不需要创建，JS会自动创建。

Math.PI = π；

Math.abs() 求得绝对值；

Math.min(1,2) // 1; 取最小值
Math.max(1,2) //2; 取最大值，两个函数都接受多个参数

Math.ceil(); 向上取整，注意正负，(-1.1) // -1;
Math.floor();向下取整
Math.round(); 四舍五入
Math.parseInt(); 取整

Math.random(); [0-1)取随机数

Math.pow(2,3); 2的3次方。可以用来取N位小数。例：

var n = Math.pow(10,n);
fixNumber = Math.round(fixNumber * n) / n;

Number对象的toFixed方法可以直接实现这个功能，5.245.toFixed(2) //5.25

Date对象

var theDate = new Date() ; 未传入参数的时候就以运行此脚本的PC的本地时间为准。

也可以直接传入时间 :

new Date(“31 January 2014”);

new Date(2014,0,31,15,35,20,20); 年月日时分秒毫秒，注意1月是0,12月是11。

保存Date对象后，可以通过一些方法取得时间

getDate() 月份中的第几天

getDay() 星期几，日是0，一是1

getMonth() 月份，一月是0

getFullYear() 返回四位数年份

toDateString() 基于当前市区，返回一个便于阅读的时间，如”Wed 31 Dec 2003”

设置日期：

setDate();月中某一天 setDate(32);会滚动到下一月
比如要知道28天后，可以

var a  = Date.getDate();
Date.setDate(a + 28); //也可以减28,获得28天前的日期

setMouth();某一月

setFullYear();某一年

一天内的日期：

getHours();setHours(); 同样可以自动滚动

getMinutes();setMinutes();

getSeconds();setSeconds();

getMilliseconds();setMilliseconds();

toTimeString(); // “13:03:51 UTC”

自定义对象

function Person (firstName,lastName)
{
    this.firstName = firstName;
    this.lastName = lastName;
}

上面的代码建立了一个原型（prototype）,注意并没有实例出对象。

new Person("Qingye","Wang"); //实例出对象

每个对象都有prototype属性，可以通过这个属性来编辑原型：

Person.prototype.getName = function(){ return this.firstName }

in和hasOwnProperty的区别是，in是只要实例上有就会返回true,而后者则必须是存在于实例上的，继承的不算。通过delete可以删除实例上的属性，还原回原型属性。
